# 미국 경기 국면 센싱 & 국면별 투자 추천 시스템 — 개발 지침 (v0.1)

> 목적: 백찬규 센터장의 매크로 로테이션 프레임워크(`youtube_report.md` + 영상 38~70분 직접 확인)를
> **재현 가능한 정량 알고리즘 + 자동 추천 엔진**으로 변환하기 위한 설계 지침.
> 이 문서는 코드가 아니라 "무엇을, 어떤 순서로, 어떤 기준으로 만들지"를 확정하는 스펙이다.

> 출처 영상: "까다로운 6월 FOMCㅣ워시의 고압 세척? 그래서 시장은 왜 빠진건데?" (NH 백찬규 센터장 출연)
> 영상 38~70분 자막 직접 확인 완료. 아래 §1.5는 영상에서 추출한 원작자의 실전 판단 레시피.

## 0. 한 줄 정의

> **매월 갱신되는 미국 매크로 데이터로 현재 경기 국면(회복/성장/둔화/침체)을 판정하고,
> 그 국면에 유리한 지수·통화·업종을 도출한 뒤, 업종 내에서 국면에 맞는 팩터로
> 개별 종목을 선별해 추천하는 신호(Signal) 시스템.**

핵심 산출물 3단: **국면 판정 → 자산/업종 배분 → 종목 숏리스트**

## 1. 설계 원칙 (리포트에서 도출한 비협상 원칙)

1. **시차 구조가 전부다.** 선행(가격) → 동행(실물) → 후행(고용/임금) 순으로 신호가 전이된다.
   국면 판정은 이 세 축의 *방향성 조합*으로 한다. 단일 지표로 판단하지 않는다.
2. **방향(모멘텀)이 레벨보다 중요하다.** 절대 수치가 아니라 **3M/6M 이동평균의 기울기**가 1순위 필터다. (리포트 §6)
3. **가짜 신호(Whipsaw) 방어가 알파의 절반이다.** 단월 급변에 리밸런싱하지 않는다. 지속성 조건을 건다.
4. **수정치(Revision) 왜곡 방지.** 고용·소매판매는 사후 대폭 수정된다. 백테스트는 반드시 **발표 시점 데이터(vintage)** 로 한다. 실시간 운영은 수정 데이터 사용.
5. **레짐 체인지 유연성.** V자 반등/급강하 가능. 신호가 충돌하면 국면을 강제로 찍지 말고 **중립(Transition)** 상태로 두고 베타를 줄인다. (리포트 §6)

## 1.5 영상 직접 확인 — 원작자의 실전 판단 레시피 (★ MVP의 뼈대)

영상 39~69분에서 백 센터장이 **"우리가 직접 판단하는 법"**을 화면으로 시연했다. 리포트의 방대한 지표 리스트보다 이 6개 압축 세트가 실전이고, 프로그램 MVP는 이걸 그대로 구현하면 된다.

### 1.5.1 실전 6대 핵심 지표 (★ 이게 코어)
| 축 | 핵심 지표 2개 | 보조 지표 | 비고 |
| :-- | :-- | :-- | :-- |
| 선행 | **주가지수**(S&P500), **제조업 PMI** | 소비자심리지수, 장단기 금리차 | PMI 50↑=확장, 영상 시점 ~55 |
| 동행 | **산업생산**, **소매판매** | 무역량(교량), 설비가동률 | 공장 가동·실제 소비량 |
| 후행 | **GDP 성장률**, **실업률** | (임금상승률·주당노동시간 = 실업률보다 더 후행) | 임금은 핵심 아님, 보통 안 봄 |

> 백 센터장 본인도 모델에서 **실업률 + GDP 성장률만** 후행축으로 본다. 임금은 "가장 늦는 지표"지만 핵심 아니라 제외. → MVP도 후행은 실업률+GDP 2개로 충분.

### 1.5.2 판단 순서 = 알고리즘 플로우 (★ 이 순서대로 구현)
영상의 진단은 거리매칭이 아니라 **순차 게이트** 방식이었다:

```
1) 선행(주가·PMI) 확인
   - 둘 다 상승 추세  → "둔화 아님" 확정, 다음으로
   - 선행이 이미 꺾임  → 둔화/침체 의심, 동행 peak 여부 확인
2) 동행(산업생산·소매판매) 확인
   - 상승         → 회복/성장 쪽
   - peak-out(꼭지 후 하락) → 둔화 진입 임박
3) 후행(GDP·실업률) 확인
   - 바닥 찍고 회복 + 실업률 여전히 높음 → 회복~성장 초입
   - GDP·실업률까지 양호(고점)         → 성장 정점
   - 실업률 상승 시작                 → 침체 전이

⇒ 영상 시점 결론: 선행↑ + 동행↑ + 후행(바닥 찍고 회복) = "성장 국면" (진입 ~10개월차)
```

**둔화 매도 트리거(원작자 직설):** "선행 지표 하락 + 동행 지표 peak-out 동시 → 있는 거 다 던져라." → 제약·유틸리티로 대피.

### 1.5.3 차트 해석·운영 규칙 (영상 그대로)
- **3~5년 차트로 추세 판단** (1년은 짧음, 10년은 다 올라보임). 실업률은 최소 3~5년.
- **12개월 이동평균(12M MA)** 으로 코로나·기저효과 노이즈 제거 후 방향 판단.
- 미국 지표는 **발표가 3~4주 늦음** → **매월 21~25일(월말/월급날) 경 1회 점검** 배치.
- 데이터는 전부 **Trading Economics(ko.tradingeconomics.com)** 에서 무료 확인. 메뉴 경로: 미국 → 주식시장 / 사업(business)>산업생산 / 소비자>연간 소매판매 / GDP 연간 / 노동>실업률.

### 1.5.4 사이클 특성 (백테스트 파라미터로 활용)
- 1바퀴(회복→성장→둔화→침체) **약 4~5.5년**, 과거보다 짧아짐.
- 국면 길이: **침체가 가장 짧음** / 성장·둔화가 상대적으로 긺 / 성장 최소 1~1.5년.
- **침체 건너뛰기 가능**: 2010년대 이후 무제한 유동성 공급으로 둔화→침체를 스킵(2020 코로나, 2022~23). → §2.2 Transition·§5 유연성의 실증 근거.

## 2. 4국면 판정 로직 (리포트 §2의 정량화)

리포트의 매트릭스를 그대로 코드로 옮긴다. 각 축은 "상승(↗)=+1 / 횡보(→)=0 / 하락(↘)=-1" 의 **방향 점수**로 환산한다.

| 국면 | 선행 | 동행 | 후행 | 스탠스 |
| :-- | :-: | :-: | :-: | :-- |
| ① 회복 Recovery | ↗ (+1) | → (0) | ↘ (-1) | Risk-On, 베타 확대 |
| ② 성장 Growth | ↗ (+1) | ↗ (+1) | ↗ (+1) | Take-Profit, 과열 비중 축소 |
| ③ 둔화 Slowdown | ↘ (-1) | ⤿ (0/-) | ↗ (+1) | 분할 매수(DCA) 시작 |
| ④ 침체 Recession | ↘ (-1) | ↘ (-1) | ↘ (-1) | 안전자산, 다음 주기 포지션 완료 |

### 2.1 각 축의 방향 점수 산출
1. 축에 속한 지표 각각을 **YoY 또는 모멘텀**으로 표준화(z-score). 코로나·기저효과 제거 위해 **12M MA** 적용(영상 §1.5.3).
2. 각 지표의 **이동평균 기울기 부호**로 +1/0/-1 라벨. (단기 모멘텀은 3M/6M MA 기울기, 추세 확인은 3~5년 차트 관점)
3. 축 점수 = 지표들의 **확산지수(Diffusion Index)** = (상승 지표 수 - 하락 지표 수) / 전체.
   - DI > +0.33 → ↗(+1), DI < -0.33 → ↘(-1), 그 사이 → →(0).
   - 영상의 **순차 게이트 플로우(§1.5.2)** 를 1차 판정 휴리스틱으로, 거리매칭(§2.2)을 신뢰도·중립 판정 보조로 병용.

### 2.2 국면 매칭 (거리 기반, 충돌 허용)
- 현재 (선행, 동행, 후행) 점수 벡터를 위 표의 4개 기준 벡터와 비교.
- **가장 가까운 국면**으로 1차 판정 + **신뢰도(confidence)** = 1 - (거리/최대거리).
- 신뢰도 < 임계값(예: 0.5) 또는 두 국면이 동률 → **Transition(중립)** 으로 표기, 베타 중립 권고.

### 2.3 침체 확정 트리거 (리포트 §5.1)
- 선행·동행이 이미 하락 반전 + **실업률이 Previous·Forecast 대비 2개월 연속 상승** → 둔화→침체 확정 전이. 안전자산 비중 최대화 플래그.

### 2.4 지속성 조건 (anti-whipsaw)
- 새 국면이 **2~3개월 연속** 유지될 때만 공식 전이 확정. 그 전엔 "잠정(provisional)" 표기.

## 3. 데이터 아키텍처 (리포트 §5 구현)

### 3.1 소스 전략 — 원본은 Trading Economics, 자동화는 FRED 권장
**원작자(백 센터장)가 영상에서 실제 사용·시연한 소스는 Trading Economics**(`ko.tradingeconomics.com`, "다 무료")다. 리포트의 실업률 지정 소스도 동일. 사람이 눈으로 보는 데는 TE가 최적(메뉴: 미국 → 주식시장/사업>산업생산/소비자>소매판매/GDP/노동>실업률).

다만 **프로그램 자동 수집**에는 TE 스크래핑이 불안정하므로, **자동화 1차 소스는 FRED(연준 공식, 무료 API, 수정이력 vintage=ALFRED)** 를 권장한다. FRED `UNRATE`는 TE 실업률과 동일 데이터를 API로 안정 제공. **TE는 Forecast(시장 예측치)·교차검증용 보조**로 사용 → §2.3 침체 트리거의 "Previous·Forecast 대비 연속 상승" 판정에 TE Forecast가 필요.

> **확정(사용자 지시 + 실측 검증, 하이브리드):** TE 무료 API는 **폐지됨**(guest 계정 HTTP 410), 차트 시계열 JSON 엔드포인트(`markets.tradingeconomics.com`)는 **DNS 차단**. 따라서 TE에서는 **(1) 현재값 (2) 예측치 `var TEForecast` (3) 차트 PNG 이미지**를 페이지 파싱으로 수집하고, **과거 시계열은 동일 원본 FRED**로 대체. 구현: `pipeline/ats/sources/tradingeconomics.py`. PMI는 과거 시계열 무료 미제공 → 선행축 모멘텀은 FRED 신규주문(NEWORDER) 프록시, PMI 현재값(50기준)은 TE 오버레이.

> ⚠️ 아래 FRED series ID는 표준 식별자 기준이며, 구현 시 각 ID 유효성/단위를 1회 검증할 것. (일부 ISM 계열은 FRED 무료 제공 중단 이력 있음 → 대체 지표 명시)

## 4. 자산/업종 로테이션 매핑 (리포트 §3·§4를 룰 테이블로)

코드가 아니라 **데이터(설정 테이블)** 로 관리한다. 룰 변경 시 코드 수정 없이 테이블만 갱신.

```
regime_playbook[국면] = {
  region:   회복=DM>EM,美>Non-US / 성장=EM선호,Non-US아웃퍼폼 / 둔화=DM>EM,美회귀 / 침체=美·DM안전쏠림
  index:    회복=NASDAQ>S&P / 성장=Russell2000 / 둔화=DOW>S&P>NASDAQ / 침체=MegaCap
  style:    회복=대형·성장 / 성장=시클리컬·중소형 / 둔화=배당·방어·가치 / 침체=퀄리티(저부채·고유동성)
  currency: 회복=USD우위 / 성장=USD약세 / 둔화=USD강세 / 침체=USD초강세
  sectors:  회복=[IT-SW, 반도체, 서비스, 커뮤니케이션, 자유소비재, 운송]
            성장=[금융, 산업재, 에너지, 소재, 리츠, 미디어&엔터, IT-HW, 헬스케어(바이오/의료기기)]
            둔화=[통신, 헬스케어(제약), 필수소비재, 유틸리티]
            침체=[MegaCap핵심주, 유틸리티, 통신]

```

## 5. 종목 선별 로직 (업종 → 개별 종목)

2단계: **(A) 국면별 유리 업종 확정 → (B) 업종 내 국면 적합 팩터로 랭킹.**

| 국면 | 종목 선별 팩터 (업종 내 스크리닝) |
| :-- | :-- |
| 회복 | 고성장·어닝 모멘텀·대형주 (Growth + Momentum) | 예: NVIDIA, M7 (반도체·IT-SW가 회복 선행) |
| 성장 | 경기민감·중소형 베타·매출성장 (Cyclical + Size) | 예: 금융·산업재·에너지·소재, 한국 반도체/2차전지/전력기기(미국 성장→이머징 주문) |
| 둔화 | 고배당·낮은 변동성·이익 가시성 (Dividend + Low-Vol + Quality) | 예: 제약(Merck), 통신(SKT), 필수소비재, 유틸리티 |
| 침체 | 저부채·고유동성·초대형 퀄리티 (Quality + Mega Cap) | 예: M7급 저부채·꾸준한 매출 메가캡, 유틸리티, 통신 |

> 위 "예시"는 영상에서 원작자가 직접 든 사례(둔화기 Merck 환+배당 20%+ 수익 등). 종목 자동선별 로직의 검증 기준점으로 활용.

- **MVP는 ETF까지만** (XLK 등 섹터 ETF + 스타일 ETF) — 검증이 쉽고 신호 품질부터 확인.
- **v2에서 개별 종목** — 섹터 ETF 구성종목 유니버스에 팩터 스코어(밸류/모멘텀/퀄리티/배당) 랭킹.
- 펀더멘털 데이터 소스(개별 종목 팩터)는 v2 진입 시 결정 (yfinance fundamentals / FMP / Alpha Vantage).

### 5.1 종목 선별 엔진 연동 — trading_america 참조 (✅ 구현)
별도 프로젝트 `trading_america` 가 이미 **S&P500 펀더멘털 3팩터 스코어링**(value/momentum/dividend 각 0~100 → 가중합, 사전필터[Altman Z·부채·이자보상], 섹터 분산 select_top)을 보유. 이를 종목 선별 단계의 정식 엔진으로 참조한다.

- **분업**: america_trend_sensing 이 **매크로 4국면 + 유리 섹터**를 결정 → 그 국면의 **전략 가중치(V/M/D)** 를 TA `score_all(df, strategy_weights=...)` 에 주입 → 펀더멘털 랭킹으로 개별종목 선별.
- **다리(bridge)**: `config/universe.json > regime_to_strategy_weights` (회복/성장=모멘텀↑, 둔화/침체=가치·배당↑). TA 의 coarse Bull/Bear regime 대신 우리의 4국면이 가중치를 공급.
- **어댑터**: `ats/stockpicker_ta.py` (`pick(regime, tickers, as_of)`). live=TA `fetch_ticker`, backtest=`build_feature_row`(PIT). TA 상대경로 때문에 호출 중 CWD 를 TA 로 고정. 미존재 시 모멘텀 폴백.
- **호출**: `recommend(use_ta=True)` / CLI `recommend --ta`. 기본은 빠른 6M 모멘텀, `--ta` 는 펀더멘털 정밀(느림).

## 7. 기술 스택 / 모듈 구조 (사용자 스택 반영)

매크로/시장 데이터 생태계는 Python이 압도적이므로 **하이브리드** 권장:

```
[수집·연산 계층]  Python (FRED/yfinance ETL + 국면 판정 + 백테스트) → MariaDB 적재
        │
[API 계층]       NestJS + TypeORM (regime/recommendation 조회 API, 스케줄러)
        │
[표현 계층]      Next.js + React 대시보드
```

- Python을 쓰기 싫으면 전부 Node로도 가능하나, FRED/시계열·백테스트 라이브러리 성숙도 때문에 수집·연산만 Python 권장. → **결정 필요 (§9)**
- 데이터 갱신은 월초(실업률 발표일 직후) 배치 + 금리차/주가 일간 배치.

## 8. 개발 로드맵 (단계별)

- **Phase 0 — 스펙 확정 (현재 문서).** 지표 목록·소스·룰 테이블 동결. ✅
- **Phase 1 — 데이터 파이프라인.** ✅ `pipeline/` 구현 완료. FRED(과거)+TE(현재값/예측/차트)+yahoo(ETF) → DB(SQLite/MariaDB). 13개 매크로 시계열·실업률/PMI TE 오버레이·15개 ETF 자동 적재.
- **Phase 2 — 국면 판정 엔진.** ✅ `ats/regime.py`. DI·거리매칭·지속성(3M)·침체 트리거. **현재 판정 = 성장(Growth), 신뢰도 1.0.**
- **Phase 3 — 백테스트/검증.** ✅ 1차 완료. `evaluate`로 NBER(USREC) 자동 대조: **P 0.42 / R 0.71 / F1 0.53**, 2001(-8M)·2008(+5M)·2020(+1M) 포착, 휘프소 0.30→0.05. **2024 오신호 해결**(레벨 게이트). 신뢰도 softmax 연속화, 침체 트리거 이원화(실업률 OR HY스프레드), 비대칭 지속성. *남은 과제: vintage(ALFRED) 무편향 백테스트, 파라미터 grid-search, 추천 알파 검증.*
- **Phase 3.5 — 전략 백테스트(E·F).** ✅ `ats/strategy.py`. 국면별 베타(현금비중)+SPY 10M MA 추세필터(서킷브레이커, 자동 재진입)+양모멘텀 섹터, 거래비용 차감 → vs SPY. **결론: 총수익 열위·드로다운 절반 → TAA 방어 오버레이로 포지셔닝.** *남은 것: vintage 백테스트(G), 환/세금, 파라미터 튜닝.*
- **Phase 4.5 — 지수 자산배분 바스켓 + 종목 과열(투자전문가 권고).** ✅ 지수층을 국면별 **자산배분 바스켓**(미국주식+지역 EFA/EEM+안전자산 TLT/GLD/SHY, 목표비중·200DMA·모멘텀)으로 — **총 주식비중**으로 모델 핵심(위험노출 타이밍) 노출. 종목에 **과열(200DMA 이격도·52주위치·RSI)** 반영: 경고 배지 + 성장(Take-Profit) 국면 **과열강도 비례 감점**. *남은 것(전문가 권고): VIX 변동성 게이트, 환(USD/KRW) 레이어, 리밸런싱 규율.*
- **Phase 4 — 추천 엔진.** ✅ `ats/recommend.py`. 현재 국면 → 플레이북(지수/스타일/통화) + **섹터 ETF 6M 모멘텀 랭킹** + **S&P500 개별종목 팩터 랭킹**(국면별 momentum/low_vol) + 관련 ETF 매핑. `recommendation` 테이블 저장. *남은 것: 알림.*
- **Phase 5 — 개별 종목(v2).** ✅ 섹터 일관 + 펀더멘털 + 시총 반영. 유니버스=현재 국면 favored 섹터 중 **양(+)모멘텀 상위 섹터**(패널과 일관) → 모멘텀 프리필터 → **trading_america 펀더멘털 3팩터(V/M/D)** + **국면별 시가총액 틸트**(회복/둔화/침체=대형 선호) + 섹터분산. 시총·ROE·V/M/D 표시. *남은 것: 둔화/침체 방어주 정밀화, 배당수익률 데이터 품질(yfinance .info 부정확) 보강.*
- **Phase 6 — 대시보드.** Next.js 시각화 + NestJS API(regime_snapshot/recommendation 서빙).

## 10. 전문가 코멘트 (리포트에 더할 점)

- 이 프레임워크는 본질적으로 **Investment Clock + 섹터 로테이션**의 변형이며 학술·실무적으로 검증된 계열이다. 약점은 **시차로 인한 진입 지연**과 **전이 구간 오판**. → §2.4 지속성 + §2.2 Transition 중립으로 방어.
- 실업률을 후행 앵커로 둔 건 타당하나, **단독 침체 트리거로 쓰면 늦다.** 선행/동행이 먼저 꺾인 *맥락에서만* 실업률 상승을 침체 확정 신호로 해석하도록 §2.3에 조건을 걸었다.
- 백테스트(Phase 3)를 통과하지 못하면 추천 엔진은 의미 없다. **개발 순서에서 Phase 3를 절대 건너뛰지 말 것.**
- **[검증 완료, 핵심 결론] 전략 백테스트(`strategy`, 2006~2026, 6개 섹터선택 변형 비교):**
  1. 국면 로테이션은 SPY 총수익을 못 이긴다(CAGR 최대 6.2% vs 11.1%). **그러나 최대낙폭을 절반으로**(-51%→-21%). → 가치는 *초과수익이 아니라 하방 방어*.
  2. **섹터선택은 알파 없음** — 상대강도·모멘텀가중·전체모멘텀 등 어떤 변형도 'SPY를 국면 베타로만 타는 것'(Sharpe 0.71)보다 못함. 회전율만 3배. → **모델 가치는 "어떤 섹터"가 아니라 "주식비중 타이밍".**
  3. → **단독 매매신호 ❌, TAA 방어 다이얼(주식비중 오버레이) ✅.** 섹터/종목 추천은 보조(정보·틸트)로 위치. (과적합 회피 위해 결과 그대로 채택)
- 영상에서 확인된 사이클 실증치(§1.5.4)를 백테스트·국면지속 파라미터의 사전값으로 사용: 1주기 4~5.5년, 침체 최단·성장/둔화 최장, 성장 최소 1~1.5년. **침체 스킵(2020·2022~23)** 사례가 있으므로 4국면 순차 전이를 강제하지 말 것.
- 후행축은 실업률+GDP로 충분(원작자도 임금은 제외). 단 §2.3 침체 확정에는 실업률의 Previous·Forecast 대비 연속 상승이 필수 → TE Forecast 데이터 확보 필요.
